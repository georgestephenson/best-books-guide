# ADR-0001 — Record architecture decisions

_Status: Accepted · 2026-07-11_

## Context
Significant technical choices need their reasoning preserved — for future contributors, for future-us, and because "why didn't you just use X?" deserves a written answer.

## Decision
Keep Architecture Decision Records in `docs/adr/`, numbered sequentially, in this lightweight format (Status/Context/Decision/Consequences). An ADR is written when a decision is hard to reverse, contrarian, or repeatedly questioned. Superseded ADRs are marked, never deleted.

## Consequences
Small writing tax per decision; in exchange, the repo explains itself. The docs index links here; CLAUDE.md instructs AI sessions to follow the same discipline.
